      [本子系翻譯] 2014-08-26

      Post on 8月 26 2014

      那麼到了今天放出翻譯題目的時候了。

      第一題是來自於 JOI14 本選的一道題目。

      綜合難度：1.5
      思考難度：2.2
      編程難度：1

      飛袋鼠

      在飛袋鼠 JOI 君居住的森林裡有 N 棵樹木，第 i 棵樹的高度為 Hi 。 JOI 君可以進行 M 種飛躍，第 i 種跳躍飛躍會從 Ai 號樹飛躍到 Bi 號樹，花費 Ti 的時間。
      當 JOI 君從高度 h 起跳并花費了 t 的時間后，他會降落在 h-t 的高度。需要注意的是 h-t 不能為負。
      JOI 君還可以爬樹，他可以花費 1 時間使高度增加或減少 1 。
      JOI 君最初在 1 號樹的 X 高度上。
      給定所有樹木的高度和所有可以進行的飛躍，和 JOI 君的初始高度，問 JOI 君跳到第 N 號樹的最頂端最少所需的時間。

      2 ≤ N ≤ 100 000．
      1 ≤ M ≤ 300 000．
      1 ≤ Hi ≤ 1 000 000 000．
      1 ≤ Ti ≤ 1 000 000 000．
      0 ≤ X ≤ H1．

      第二道題是來自 COCI14 R6 的一道題目。

      綜合難度：1.2
      思考難度：1.7
      編程難度：1

      哈希

      我們定義這樣一個哈希函數。

      f(empty word)=0
      f(word+letter)=(f(word)*33xor ord(letter))%MOD
      ord(letter)=letter-‘a’+1

      現在我們希望知道長度為 N 的字符串中 MOD＝2^M 時哈希值為 K 的字符串的個數。

      1 ≤ N ≤ 10
      0 ≤ K < 2^M
      6 ≤ M ≤ 25

      飛袋鼠解法

      對於小課題 1 ，顯然可以用最短路來解決。
      對於每一棵樹的每一個高度建一個點，再用 Dijkstra 演算法解決，可以解決小課題 1 。
      時間複雜度 O(N(H+M)log(NH)) 。

      對於小課題 2 ，我們注意到我們不需關心高度，每次只要爬 cost(u, v) 高度再飛躍過去就可以，所以也規約到了最短路問題。
      時間複雜度 O((N+M)logN) 。

      這道題的滿點解法就是把小課題 2 的解法推廣到一般情況。
      我們首先發現可以把 h-t 非負的約束轉化為 Hu-t 非負的約束( u 是當前所在的樹)，這樣我們就發現我們不需要考慮高度的問題了。
      也就是說，我們的飛躍其實可以看做是下降了一段距離后再爬上來。
      假設我們飛躍到了 N 號樹，我們在 h 的高度，那麼答案就是 (X-h)+(HN-h) 。
      同樣用 Dijkstra 演算法解決，時間複雜度 O((N+M)logN) 。

      解答例：https://www.dropbox.com/s/gesfh30zkecedqs/2014-ho-t4.cpp?dl=0

      哈希解法

      我們可以發現，它只需要求一種哈希值的結果。
      正解使用了雙向搜索法。
      我們可以發現，我們只需要知道前一半字符串的哈希值，就可以得出后一半字符串的哈希值。
      而這一過程的時間複雜度是 O(1) 的。
      當然還涉及到了求逆元。
      因為具體實現搜索的時候，前一半很好搜索，對於後一半我們可以從后往前搜索，也就是倒退回去，這當然就要用到逆元了。

      解答例：https://www.dropbox.com/s/ol49masdejyvedx/hash.cpp?dl=0
